# 分批生成方案：实现 50 页 PPT

> 针对日本军工企业深度研究 PDF 的分批生成方案

---

## 方案概览

| 项目 | 配置 |
|------|------|
| **目标总页数** | 50 页 |
| **分批数** | 4 批 |
| **每批页数** | 12-13 页 |
| **源文档** | 日本军工企业深度研究 (2).pdf |

---

## 分批策略

### 章节分配

原文档共 7 个章节，按逻辑分为 4 批：

| 批次 | 章节 | 内容主题 | 目标页数 |
|------|------|----------|----------|
| **批次 1** | ch01 + ch02 | 宏观政策与巨头矩阵 | 12-13 页 |
| **批次 2** | ch03 + ch04 | 电子技术与大鲸级潜艇 | 12-13 页 |
| **批次 3** | ch05 + ch06 | 坦克与航空动力 | 12-13 页 |
| **批次 4** | ch07 | 全球战略与结论 | 12-13 页 |

### 生成流程

```
原始 PDF
    ↓
Step 0: 内容提取（1次）
    ↓
Step 1: 结构化分析（1次，7章完整分析）
    ↓
    ├─→ 批次 1 (ch01-02) → 生成 12页大纲 → 生成视觉资产 → 生成 PPT
    ├─→ 批次 2 (ch03-04) → 生成 12页大纲 → 生成视觉资产 → 生成 PPT
    ├─→ 批次 3 (ch05-06) → 生成 12页大纲 → 生成视觉资产 → 生成 PPT
    └─→ 批次 4 (ch07)   → 生成 12页大纲 → 生成视觉资产 → 生成 PPT
    ↓
合并所有 PPT（添加批次分隔页）
    ↓
最终输出: 50页完整 PPT
```

## 技术实现

### 方法 1: 章节拆分法（推荐）

```python
# 伪代码
chapters = analysis['chapters']  # 7章

# 分配章节给各批次
batch1_chapters = chapters[0:2]   # ch01-02: 宏观政策+巨头
batch2_chapters = chapters[2:4]   # ch03-04: 电子技术+潜艇
batch3_chapters = chapters[4:6]   # ch05-06: 坦克+航空
batch4_chapters = chapters[6:7]   # ch07: 全球战略

# 为每批生成独立大纲
for batch in [batch1, batch2, batch3, batch4]:
    batch_analysis = {
        **analysis,
        'chapters': batch['chapters'],
        'context': f"这是第{batch['id']}部分，共4部分"
    }
    
    # 生成该批次的 slide_plan（12-13页）
    slide_plan = generate_outline(batch_analysis, slides=12)
    
    # 生成视觉资产
    assets = generate_visuals(slide_plan)
    
    # 生成 PPT
    pptx = build_pptx(slide_plan, assets)
```

## 优势分析

| 优势 | 说明 |
|------|------|
| **质量可控** | 每批12页，LLM 能充分展开每页内容 |
| **逻辑连贯** | 按章节拆分，保证批次内逻辑完整 |
| **并行生成** | 4批可并行处理（如果API允许） |
| **容错性高** | 单批失败可重试，不影响其他批次 |
| **易于审查** | 每批可单独审查后再合并 |

## 注意事项

1. **API 限制**
   - 注意 LLM API 的速率限制
   - 4 批连续生成可能需要 10-15 分钟

2. **Token 消耗**
   - 每批生成约消耗 15K-20K tokens
   - 总计约 60K-80K tokens

3. **内容重复**
   - 批次间需避免内容重复
   - 通过章节边界自然分隔

4. **视觉一致性**
   - 所有批次使用相同主题
   - 合并时统一配色和字体
