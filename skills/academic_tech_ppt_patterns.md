## 二、页面分类体系（10种核心类型）

### 1. COVER（封面页）
- **出现位置**：第1页
- **特征**：大标题（32-66pt），副标题/单位/日期，背景图或装饰图形
- **文字量**：40-200字符
- **形状数**：3-9个
- **关键元素**：项目全称、单位logo、日期、主视觉图
- **标题字号**：32-66pt（学术类32-48pt，答辩类60-66pt）
- **设计要点**：
  - 标题居中或左对齐，占页面上半部
  - 背景色深蓝(#002060/#092E56)或渐变
  - 底部条带放单位名称和日期
  - 可有1-3个装饰性背景图

### 2. SECTION（章节分隔页）
- **出现频率**：每PPT 3-6个，占总页数10-15%
- **特征**：极少文字（<30字符），大字号章节编号
- **文字量**：8-29字符
- **形状数**：3-11个
- **关键元素**：章节编号（1/2/3或"一/二/三"）、章节标题
- **标题字号**：32-96pt
- **设计要点**：
  - 编号特大（60-96pt），粗体
  - 标题简洁，一行以内
  - 背景可用纯色/渐变/图片+遮罩
  - 视觉效果突出，与内容页形成明确分隔
  - 部分使用圆形编号徽章

### 3. IMG+TEXT（图文混排页）⭐ 最高频
- **出现频率**：占总页数40-60%，是主力页面类型
- **特征**：1-14张图片 + 50-600字符文字
- **形状数**：4-64个
- **关键元素**：系统架构图、流程图、截图、示意图 + 文字说明
- **标题字号**：24-36pt
- **正文字号**：14-20pt
- **设计要点**：
  - 图片占页面50-70%面积
  - 文字为要点式说明（非大段落）
  - 常见布局：左图右文、上图下文、中心图+四周说明
  - 图片多为系统截图、架构图、流程图
  - 标题在顶部，底部可有总结行

### 4. TEXT-HEAVY（文字密集页）
- **出现频率**：占总页数10-15%
- **特征**：文字>200字符，无图片或仅有装饰图
- **文字量**：200-1000+字符
- **形状数**：5-79个（含大量文本框和装饰形状）
- **关键元素**：多级要点列表、技术描述、背景分析
- **标题字号**：28-60pt
- **正文字号**：16-24pt
- **设计要点**：
  - 使用多级项目符号（●/▸/—）
  - 关键数据用红色(#FF0000/#C00000)加粗
  - 每条要点15-40字
  - 可用色块/卡片分组
  - 避免纯文字墙，用装饰线条分隔

### 5. IMAGE-HEAVY（图片主导页）
- **出现频率**：占总页数8-15%
- **特征**：3+张图片，文字<100字符
- **文字量**：0-96字符
- **形状数**：5-20个
- **关键元素**：产品照片、实验结果图、系统界面截图
- **设计要点**：
  - 图片矩阵布局（2×2, 3×2, 1大+2小）
  - 极简文字标注
  - 图片边框统一（圆角或阴影）
  - 适用于展示成果、设备、界面

### 6. TABLE（表格页）
- **出现频率**：占总页数5-10%
- **特征**：含表格元素
- **关键元素**：数据对比表、参数表、进度表
- **设计要点**：
  - 表头深色背景（#002060/#0060A8）白字
  - 奇偶行交替浅色背景
  - 字号12-16pt
  - 表格占页面60-80%宽度
  - 标题在上方，表格下方可有注释

### 7. CHART（图表页）
- **出现频率**：占总页数0-5%（学术PPT中图表多以图片形式嵌入）
- **特征**：原生图表对象
- **关键元素**：柱状图、折线图、饼图
- **设计要点**：
  - 学术PPT更常用matplotlib/截图嵌入而非原生图表
  - 颜色与整体配色方案一致
  - 数据标签清晰

### 8. CONTENT（内容过渡页）
- **出现频率**：占总页数5-15%
- **特征**：中等文字量，无图片，可能是目录/提纲
- **文字量**：0-170字符
- **关键元素**：大纲列表、章节预览、过渡说明
- **设计要点**：
  - 标题突出（32pt+）
  - 内容简洁
  - 可用图标装饰

### 9. END（结束页）
- **出现位置**：最后一页
- **特征**：极简文字（"谢谢！""汇报完毕"）
- **文字量**：3-22字符
- **标题字号**：40-115pt
- **设计要点**：
  - "谢谢" / "感谢聆听" 超大字
  - 与封面风格呼应
  - 可加联系方式

### 10. METHODOLOGY（方法论页，学术特有）
- **出现频率**：学术PPT中占10-20%
- **特征**：研究内容分步骤展示，编号式标题
- **关键元素**：
  - 标题格式："研究内容(一)：XXX" / "创新点(1)：XXX"
  - 技术路线图/算法流程
  - 公式或伪代码
- **设计要点**：
  - 编号一致性（中文/阿拉伯/罗马）
  - 流程图箭头清晰
  - 数学符号使用Times New Roman

## 三、配色方案

### 主色调（按频率排序）

| 色值 | 用途 | 出现频率 |
|------|------|----------|
| `#002060` | 深海蓝 - 标题、主要文字 | ★★★★★ 所有PPT |
| `#C00000` | 暗红色 - 强调、关键数据 | ★★★★★ 所有PPT |
| `#FF0000` | 纯红 - 重点标注 | ★★★★ 5/6 PPT |
| `#0060A8` | 科技蓝 - 形状填充、图标 | ★★★★ 4/6 PPT |
| `#FFFFFF` | 白色 - 深色背景上的文字 | ★★★★ 4/6 PPT |
| `#178F95` | 青绿色 - 辅助色、图标 | ★★★ 3/6 PPT |
| `#4472C4` | Office蓝 - 图表、装饰 | ★★★ 3/6 PPT |
| `#FFC000` | 金黄色 - 高亮、徽章 | ★★ 2/6 PPT |
| `#70AD47` | 绿色 - 正向指标 | ★★ 2/6 PPT |
| `#092E56` | 极深蓝 - 背景、侧边栏 | ★★ 2/6 PPT |

### 推荐配色方案

```json
{
  "academic_tech": {
    "primary": "#002060",
    "secondary": "#0060A8",
    "accent": "#C00000",
    "highlight": "#FF0000",
    "success": "#178F95",
    "warning": "#FFC000",
    "background": "#FFFFFF",
    "text": "#333333",
    "text_light": "#FFFFFF"
  }
}
```

### 配色使用规则
1. **标题文字**：`#002060`（深海蓝），粗体
2. **关键数据/强调**：`#C00000`或`#FF0000`，粗体
3. **正文文字**：`#333333`或`#000000`
4. **形状填充**：`#0060A8`（科技蓝）为主，`#178F95`（青绿）为辅
5. **背景**：白色为主，章节页可用`#092E56`（极深蓝）
6. **图表色**：`#4472C4` → `#ED7D31` → `#70AD47` → `#FFC000`

|-----|--------|-----------|------------|
| A (储能平台) | 208 | 100% | 5.5 |

| C (图理论BM) | 66 | 59% | 2.5 |

| E (输配智囊) | 48 | 50% | 6.9 |
| F (数据治理) | 36 | 97% | 1.2 |

### 常见图片类型
1. **系统架构图**：技术方案的核心，通常占半页以上
2. **流程图**：研究方法、算法步骤
3. **系统界面截图**：产品展示、实验结果
4. **实物照片**：设备、实验场景
5. **示意图/图标**：概念说明
6. **公式/数学推导**：学术类特有
7. **团队照片/logo**：项目申报类

### 装饰性形状
- **AUTO_SHAPE 占比最高**：圆角矩形、箭头、色块
- **LINE（线条）**：分隔线、连接线、装饰线
- **GROUP（组合）**：复杂图形组合体
- **FREEFORM（自由形状）**：自定义箭头、流程连接

### 形状使用密度
| PPT类型 | 平均形状/页 |
|---------|------------|
| 科技项目申报 | 14-16 |
| 学术答辩 | 12-14 |
| 产品演示 | 15-20 |
| 技术方案 | 8-10 |

## 六、页面布局模式

### 模式A：左图右文（最常见）
```
┌─────────────────────────────────┐
│  [标题]                          │
├────────────────┬────────────────┤
│                │  • 要点1        │
│   架构图/      │  • 要点2        │
│   流程图       │  • 要点3        │
│                │  • 要点4        │
├────────────────┴────────────────┤
│  [底部总结/来源]                  │
└─────────────────────────────────┘
```

### 模式B：上图下文
```
┌─────────────────────────────────┐
│  [标题]                          │
├─────────────────────────────────┤
│       系统架构图/技术路线图        │
│                                  │
├─────────────────────────────────┤
│  • 要点1  │  • 要点2  │  • 要点3 │
└─────────────────────────────────┘
```

### 模式C：多图矩阵
```
┌─────────────────────────────────┐
│  [标题]                          │
├────────┬────────┬───────────────┤
│ [图1]  │ [图2]  │               │
│ 说明1  │ 说明2  │   [大图/      │
├────────┼────────┤    说明]      │
│ [图3]  │ [图4]  │               │
│ 说明3  │ 说明4  │               │
└────────┴────────┴───────────────┘
```

### 模式D：中心辐射（技术架构常用）
```
┌─────────────────────────────────┐
│  [标题]                          │
│                                  │
│     [模块1]  ←→  [核心]  ←→  [模块2] │
│                ↕                 │
│     [模块3]  ←→  [模块4]        │
│                                  │
│  [底部说明]                       │
└─────────────────────────────────┘
```

### 模式E：编号分步（方法论页面）
```
┌─────────────────────────────────┐
│  研究内容(一)：XXXX               │
├─────────────────────────────────┤
│  ①           ②           ③     │
│ [步骤1]    [步骤2]     [步骤3]   │
│  说明        说明        说明    │
│    ──→         ──→              │
├─────────────────────────────────┤
│  [技术路线图/流程图]              │
└─────────────────────────────────┘
```

## 七、学术/科技PPT特有模式

### 1. 项目申报类结构
```
封面 → 目录 →
[第一章] 项目背景与意义 → 政策背景 → 行业现状 →
[第二章] 研究基础 → 团队介绍 → 已有成果 →
[第三章] 研究方案 → 技术路线 → 研究内容1-N →
[第四章] 预期成果 → 考核指标 →
[第五章] 经费预算 → 进度安排 →
结束页
```
- 页数：30-60页
- 特点：内容详实，图文并茂，数据支撑

### 2. 学术答辩类结构
```
封面 → 团队介绍 →
[第一部分] 研究背景 → 文献综述 →
[第二部分] 研究方法 → 技术方案 → 算法流程 →
[第三部分] 实验结果 → 性能对比 →
[第四部分] 应用场景 → 案例展示 →
[第五部分] 未来工作 →
结束页
```
- 页数：14-44页
- 特点：逻辑严密，方法论清晰

### 3. 技术方案类结构
```
封面 →
[背景] 现状分析 → 痛点问题 →
[方案] 总体架构 → 技术路线 → 各模块详解 →
[验证] 实验/测试 → 结果分析 →
[总结] 核心优势 → 下一步计划 →
结束页
```
- 页数：20-40页
- 特点：架构图核心，模块化展示

### 4. 关键差异化元素
| 元素 | 项目申报 | 学术答辩 | 技术方案 |
|------|---------|---------|---------|
| 团队介绍 | ✅必须 | ✅简要 | ❌不需要 |
| 经费预算 | ✅必须 | ❌不需要 | ❌不需要 |
| 公式推导 | 可选 | ✅核心 | 可选 |
| 代码展示 | ❌少见 | 可选 | ✅常见 |
| 政策引用 | ✅必须 | 可选 | 可选 |
| 性能对比表 | 可选 | ✅核心 | ✅核心 |
| 实物照片 | ✅多 | 可选 | 可选 |
| 系统截图 | 可选 | 可选 | ✅多 |

## 八、内容密度标准

### 每页文字量建议
| 页面类型 | 建议字符数 | 参考范围 |
|---------|-----------|---------|
| 封面 | 40-100 | 项目名+单位+日期 |
| 章节页 | 10-30 | 编号+标题 |
| 图文页 | 100-300 | 3-5条要点+图 |
| 文字页 | 200-500 | 5-8条要点 |
| 表格页 | 50-200 | 标题+表格+注释 |
| 结束页 | 5-25 | 致谢语 |

### 图文比例建议
- **图片主导页**：图70% + 文30%
- **图文混排页**：图50% + 文50%
- **文字密集页**：文80% + 装饰20%
- **整体PPT**：每页平均1-3张图片/图表

## 九、生成策略建议

### 针对Super-PPT系统的优化方向

1. **新增页面类型**：
   - `methodology`：研究方法/技术路线页
   - `team`：团队介绍页
   - `architecture`：系统架构图页（中心+辐射）
   - `comparison_table`：性能对比表页
   - `timeline_progress`：进度/里程碑页

2. **配色方案扩展**：
   - 新增 `academic` 配色（#002060/#0060A8/#C00000）
   - 新增 `tech_report` 配色（#092E56/#178F95/#ED7D31）

3. **字体规范**：
   - 中文统一微软雅黑
   - 英文/数字用Times New Roman或Calibri
   - 强调用粗体+红色，而非仅加粗

4. **布局优化**：
   - 学术类优先"左图右文"布局
   - 架构图页使用全宽布局
   - 方法论页使用编号分步布局

5. **视觉元素增强**：
   - 增加装饰性线条（分隔线、连接线）
   - 标题区底部加蓝色/红色装饰条
   - 色块分组（浅蓝底+深蓝边框）
   - 编号使用圆形徽章样式

6. **内容标准**：
   - 每页要点3-5条，每条15-40字
   - 关键数据必须红色加粗
   - 章节编号统一格式
   - 底部统一加来源/注释区域
