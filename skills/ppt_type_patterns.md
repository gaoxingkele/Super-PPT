# PPT类型设计模式库

> 8种核心PPT类型的设计规则，供 Step1 类型识别和 Step2 大纲设计使用。

---

## 类型1: 学术答辩 (academic_defense)

**场景**: 毕设/硕博答辩、学术成果汇报
**核心目标**: 证明研究的价值和严谨性

### 结构模板
```
封面 → 目录 →
[PART 1] 研究背景与问题定义 → 文献综述/现状分析 →
[PART 2] 研究方法/技术方案 → 算法/模型设计 →
[PART 3] 实验设计 → 结果分析 → 性能对比 →
[PART 4] 应用场景/案例验证 →
[PART 5] 总结与展望 →
结束页
```

### 设计规则
- 白底为主，深蓝(#002060)标题，暗红(#C00000)仅用于关键强调
- 每个章节以 section_break 开头，带 PART 编号
- 优先使用 methodology 和 architecture 布局
- 必须有"问题→方法→实验→结果"完整论证链
- 性能对比必须有量化数据（表格或图表）
- 公式/算法用独立页面展示，字号不小于16pt

### 节奏规划
- 每章3-5页内容 + 1页章节过渡
- 实验结果部分可密集（连续数据页），但每3页后插1页关键发现总结
- 总页数：15-45页

## 类型2: 项目申报 (project_proposal)

**场景**: 国家级/省级课题申报、基金申请、重点项目答辩
**核心目标**: 争取评审通过和资金支持

### 结构模板
```
封面 → 目录 →
[PART 1] 项目背景与意义 → 政策背景 → 行业痛点 → 国内外现状 →
[PART 2] 研究基础 → 团队介绍 → 已有成果/平台 → 前期工作 →
[PART 3] 研究方案 → 总体技术路线 → 研究内容(1) → 研究内容(2) → ... →
[PART 4] 预期成果 → 考核指标 → 创新点总结 →
[PART 5] 经费预算 → 进度安排 →
结束页
```

### 设计规则
- 内容详实，图文并茂，数据支撑充分
- 政策引用必须有（国家战略/行业标准）
- 团队介绍页：成员照片+头衔+代表成果
- 技术路线图必须用 methodology 布局，全宽展示
- 考核指标用表格，量化具体（论文X篇、专利X项、系统X个）
- 经费和进度可用 table 或 timeline 布局
- 创新点用编号+关键词+展开描述

### 节奏规划
- 背景部分适度（3-5页），重点在方案（占50%以上）
- 研究内容每个子课题独立2-3页
- 总页数：30-60页

## 类型3: 技术方案 (technical_solution)

**场景**: 系统架构设计、工具介绍、技术选型汇报
**核心目标**: 展示方案的可行性和技术优势

### 结构模板
```
封面 → 目录 →
[背景] 现状分析 → 痛点问题 →
[方案] 总体架构 → 技术选型 → 各模块详解(1-N) →
[验证] 实验/测试 → 性能数据 → 结果分析 →
[案例] 实际部署/应用案例 →
[总结] 核心优势 → 未来规划 →
结束页
```

### 设计规则
- 架构图是核心，必须有全宽 architecture 页
- 模块详解用 title_content（左文右截图）
- 必须有工具界面截图/Demo展示
- 性能数据用 data_chart（柱状图/折线图对比）
- 痛点→方案→效果验证必须闭环
- 代码片段可用 quote 布局（等宽字体）

### 节奏规划
- 背景简短（2-3页），方案详细（占60%）
- 架构图后紧跟模块分解
- 总页数：20-40页

## 类型4: 竞赛路演 (competition_pitch)

**场景**: 创新大赛、创业路演、产品评审
**核心目标**: 说服评委/投资人，突出差异化优势

### 结构模板
```
封面（视觉冲击力强）→
[痛点] 行业痛点/用户故事（1-2页）→
[方案] 核心方案概述（1页）→ 技术亮点(2-3页) →
[壁垒] 竞争分析/差异化优势（1-2页）→
[成果] 已有成果/数据验证（1-2页）→
[团队] 核心团队（1页）→
[展望] 未来规划/商业模式（1页）→
结束页
```

### 设计规则
- 视觉冲击优先，封面和关键页用高质量generate-image
- 痛点页用数据或故事引发共鸣（不是技术描述）
- 方案概述用一张图讲清楚（信息图/架构图）
- 竞争分析用 two_column 或 radar 图对比
- 每页一个核心信息，文字极简
- 关键数字超大展示（KPI卡片）
- 总页数精简：12-20页

### 节奏规划
- 开头2页必须抓住注意力（痛点/数据冲击）
- 中间节奏紧凑，无废话页
- 每页停留时间 < 90秒

## 类型5: 行业研报 (industry_report)

**场景**: 市场分析、行业白皮书、咨询报告
**核心目标**: 提供数据驱动的决策依据

### 结构模板
```
封面 → 目录 →
[概述] 行业定义/范围 → 核心发现（3-5条）→
[市场] 市场规模 → 增长趋势 → 驱动因素 →
[格局] 竞争格局 → 头部企业分析 →
[技术] 技术趋势 → 创新方向 →
[趋势] 未来展望 → 投资建议 →
结束页
```

### 设计规则
- 数据图表为主（60%以上页面有图表）
- 每个数据图表必须配2-3条洞察bullets
- 颜色方案：primary:#1B365D, accent:#E8612D
- 关键发现页（key_insight）前置，开头就给结论
- 来源标注：每个数据图表底部注明数据来源
- 企业分析用 two_column 或 table

### 节奏规划
- 核心发现前置（先结论后论证）
- 数据页和解读页交替
- 总页数：20-40页

## 类型6: 产品演示 (product_demo)

**场景**: 产品发布、功能演示、客户培训
**核心目标**: 激发购买/使用意愿

### 结构模板
```
封面 →
[价值] 产品定位/价值主张（1页）→
[功能] 核心功能展示(3-5页) →
[场景] 应用场景/用户案例(2-3页) →
[对比] vs竞品优势（1页）→
[演示] 操作流程/Demo（2-3页）→
[定价] 方案/套餐（可选）→
结束页（CTA行动号召）
```

### 设计规则
- 产品截图是核心，大面积高清截图
- 功能展示用 title_content（左文字右截图）
- 用户场景用故事化表达（人物+场景+解决方案）
- CTA明确（免费试用/联系我们/扫码体验）
- 配色活泼但专业，可用品牌色

### 节奏规划
- 开头1页讲清"解决什么问题"
- 功能不要面面俱到，聚焦3-5个核心功能
- 总页数：12-25页

## 类型7: 单次知识讲座 (single_lecture)

**场景**: 专题培训、技术分享、公开课、学术讲座
**核心目标**: 让听众在1次课内"听懂并记住"

### 结构模板
```
封面 →
[引入] 悬念/问题/故事开场（1-2页）→
[概览] 今天要讲什么（目录页，3-5个主题）→
[主题1] 概念引入 → 核心讲解 → 案例/演示 → 小结 →
[主题2] 概念引入 → 核心讲解 → 案例/演示 → 小结 →
[主题3] 概念引入 → 核心讲解 → 案例/演示 → 小结 →
[总结] 一页纸总结（听众拍照带走）→ Q&A →
结束页
```

### 设计规则
- **教学节奏**: 概念→例子→概念→例子→小结，循环推进
- **概念拆解**: 每个新概念必须配类比/比喻/生活化案例
- **视觉锚点**: 一张图代表一个概念，反复出现强化记忆
- **小结回顾**: 每10-15页插一个"阶段小结"页（防遗忘曲线）
- **互动设计**: 适当插入"思考题"或"举手调查"页面
- **一页纸总结**: 结尾必须有浓缩全部要点的总结页
- 字号偏大（正文18-20pt），适应大教室投影
- 每页文字更少（3-4条bullet，每条20字以内）
- 避免密集数据页，用直觉化的类比替代

### 特殊页面类型
- **concept_intro**: 概念引入页 — 大标题+类比图+一句话定义
- **example_case**: 案例页 — 场景描述+实际截图/数据
- **checkpoint**: 小结页 — "到目前为止我们学了什么"（3-5条回顾）
- **interaction**: 互动页 — "思考：如果是你会怎么做？"

### 节奏规划
- 开头必须有趣（故事/数据冲击/反直觉问题）
- 每个主题内部：引入(1页)→讲解(2-3页)→案例(1-2页)→小结(1页)
- 每15分钟一个自然断点
- 总页数：25-50页（1小时讲座约35页）

## 类型8: 系列课程学习 (course_series)

**场景**: 多课时培训、系列课程、教材配套PPT
**核心目标**: 系统掌握知识体系，建立完整认知框架

### 结构模板
```
封面（含课程编号: 第X课/共Y课）→
[知识地图] 整体课程结构，高亮本课位置 →
[回顾] 上节课要点回顾（3条bullet）→
[本课目标] 本课学习目标（2-3条，具体可检验）→
[内容] 知识点1讲授 → 知识点2讲授 → ... →
  （每个知识点后配练习/思考题）→
[实践] 综合练习/案例实操 →
[总结] 本课要点总结 → 下节预告 → 课后任务 →
结束页
```

### 设计规则
- **知识地图页**: 每课开头必须有，树状/流程图展示整体知识结构，当前课程高亮，已学灰显
- **回顾衔接**: 开头用2-3条bullet回顾上节课核心要点
- **学习目标**: 用"学完本课你将能够..."句式，具体可检验
- **分级标记**: 基础内容(蓝色标签)、进阶内容(橙色标签)、拓展内容(灰色标签)
- **术语定义框**: 新术语首次出现时用带底色(#F0F4F8)的定义卡片
- **练习页**: 每2-3个知识点后插入练习/思考题页
- **前后衔接**: 结尾有"本课要点"+"下节预告"+"课后任务"三板块
- **编号体系**: 页码格式 L{课号}-P{页号}（如 L03-P12）
- 内容要有层次：定义→原理→示例→练习→总结

### 特殊页面类型
- **knowledge_map**: 知识地图页 — 树状/网络图，当前位置高亮
- **review**: 回顾页 — "上节课我们学了..."
- **definition**: 术语定义页 — 术语卡片（术语+定义+示例）
- **exercise**: 练习页 — 题目+选项/填空
- **preview**: 预告页 — "下节课我们将学习..."

### 节奏规划
- 知识地图→回顾→目标（前3页固定）
- 每个知识点：概念(1页)→原理(1-2页)→示例(1页)→练习(1页)
- 每课4-6个知识点
- 总页数：30-50页/课
- 全系列保持视觉风格完全一致

## 跨类型通用规则

### 1. 目录页规则
- 目录/提纲页最多出现1次（放在开头）
- 章节过渡用 section_break，不重复完整目录

### 2. 文字密度控制
- 每页正文不超过6行/120字
- 单条bullet 15-40字
- 超长内容拆分为多页

### 3. 视觉节奏
- 连续超过3页密集文字后必须插入视觉缓冲页
- 视觉缓冲页类型：image_full / quote / section_break / key_insight
- 数据页和解读页交替出现

### 4. 闭环论证
- 所有类型都必须有论点→论据→结论的基本闭环
- 提出问题必须有解决方案，展示方案必须有效果验证

### 5. 首尾呼应
- 封面和结束页风格统一
- 结尾回顾开头提出的核心问题/目标
